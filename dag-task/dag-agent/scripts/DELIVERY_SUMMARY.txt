📈 核心测试场景
─────────────────────────────────────────────────────────────

1️⃣  顺序提交测试 (Sequential Submission)
   - 单线程顺序提交所有任务
   - 测试基线性能
   - 易于调试

2️⃣  批量提交测试 (Batch Submission)
   - 将任务分批提交
   - 批内并发，批间串行
   - 平衡性能

3️⃣  并发提交测试 (Concurrent Submission)
   - 多线程并发提交
   - 测试最大吞吐量
   - 找出瓶颈

4️⃣  混合工作负载测试 (Mixed Workload)
   - 提交 + 健康检查
   - 模拟真实场景
   - 全面评估


💡 核心指标解读
─────────────────────────────────────────────────────────────

吞吐量 (Throughput)
  ├─ 定义: 单位时间处理的任务数
  ├─ 单位: tasks/second
  ├─ 期望: 顺序 >1000, 并发 >2000
  └─ 例: "40.16 tasks/sec"

延迟 (Latency)
  ├─ 定义: 请求响应时间
  ├─ 关键: P95 和 P99 (实际用户体验)
  ├─ 期望: P95 <10ms, P99 <50ms
  └─ 例: "P95: 8.3ms, P99: 45.2ms"

成功率 (Success Rate)
  ├─ 定义: 成功提交占比
  ├─ 单位: 百分比 (%)
  ├─ 期望: >99.5%
  └─ 例: "Success Rate: 99.95%"

⚙️ 参数配置速查
─────────────────────────────────────────────────────────────

benchmark-quick.sh
  --url <url>         Agent URL (default: http://localhost:20000)
  --tasks <n>         任务数量 (默认: 50)
  --threads <n>       并发线程数 (默认: 4)

benchmark-performance-test.sh
  --url <url>         Agent URL
  --tasks <n>         任务数量 (默认: 100)
  --threads <n>       并发线程数 (默认: 4)
  --batch-size <n>    批处理大小 (默认: 10)
  --task-type <type>  任务类型: echo|shell|io|cpu (默认: echo)
  --duration <sec>    最大测试时长 (默认: 300)
  --help              显示帮助


⚠️ 常见问题
─────────────────────────────────────────────────────────────

Q: 连接超时？
A: 1. 检查: curl http://localhost:20000/agent/api/v1/agent/ping
   2. 如失败，启动Agent: cd dag-agent-muserver && mvn exec:java -Denv=local

Q: 成功率低？
A: 减少负载: ./benchmark-quick.sh --tasks 30 --threads 2

Q: 延迟很高？
A: 使用轻量级任务: ./benchmark-performance-test.sh --task-type echo

Q: Python分析工具找不到？
A: 检查Python安装: python3 --version
   或使用绝对路径: python3 /path/to/benchmark-analyzer.py

更多问题见 BENCHMARK_README.md 的故障排查部分
