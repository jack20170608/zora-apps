### 2.5 端到端测试场景

#### 场景 1: 简单线性 DAG

```
Task A → Task B → Task C
```

**测试步骤**:
1. 创建任务模板 A, B, C
2. 创建 DAG 定义依赖 A → B → C
3. 触发 DAG 执行
4. 验证 A → B → C 顺序执行
5. 验证最终所有任务成功

**预期结果**: 严格按依赖顺序执行

#### 场景 2: 分叉汇聚 DAG

```
    ┌→ B ─┐
A ──┤     ├→ D
    └→ C ─┘
```

**测试步骤**:
1. 创建四个任务模板
2. 创建依赖关系
3. 触发执行
4. 验证 B 和 C 可以并行执行
5. 验证 B 和 C 都完成后 D 才开始执行

**预期结果**: B/C 并行，D 最后执行

#### 场景 3: DAG 循环检测

**测试步骤**:
1. 创建带有循环的 DAG (A→B→C→A)
2. 验证创建被拒绝
3. 验证返回明确的循环错误信息

**预期结果**: 创建失败，提示循环依赖

#### 场景 4: 手动操作 - hold/free

**测试步骤**:
1. 创建一个长时间运行的任务
2. 任务执行中执行 hold 操作
3. 验证任务被中断
4. 执行 free 操作
5. 验证任务被重新调度

**预期结果**: hold 正确暂停，free 正确恢复

#### 场景 5: 手动操作 - forceOk/forceNok

**测试步骤**:
1. 创建一个卡住的长时间运行任务
2. 执行 forceOk / forceNok
3. 验证任务被强制标记完成
4. 验证后继任务正确触发

**预期结果**: 强制完成成功，后继任务正确执行

#### 场景 6: 多 Agent 负载均衡

**测试步骤**:
1. 启动 3 个 Agent
2. 批量提交 10 个任务
3. 验证任务按负载均衡策略分发

**预期结果**: 负载均衡策略生效

#### 场景 7: 死信队列重试

**测试步骤**:
1. Agent 断开网络
2. 分发任务给该 Agent
3. 结果上报失败，进入死信队列
5. Agent 恢复网络
6. 验证自动重试成功

**预期结果**: 自动重试，最终上报成功

#### 场景 8: 背压测试

**测试步骤**:
1. 配置 Agent: maxPending = 2
2. 提交 3 个任务
3. 验证前 2 个接受，第 3 个返回 429

**预期结果**: 正确的容量控制，429 响应

### 3.2 压力测试场景

| 场景 | 测试指标 | 目标 SLA |
|------|----------|----------|
| **小负载** - 100 个任务，DAG 深度 5 | 完成时间 | < 30 秒 |
| **中负载** - 1000 个任务，并发 10 | 完成时间 | < 5 分钟 |
| **大负载** - 10000 个任务 | 系统稳定性 | 无 OOM，无数据损坏 |
| **持续负载** - 每小时 1000 任务，持续 24 小时 | 内存泄漏 | 堆内存稳定，无持续增长 |
| **并发测试** - 多个调度线程同时调度 | 数据一致性 | 无重复执行，无状态错乱 |

### 3.3 性能监控指标

需要监控:
- 调度延迟 (从任务就绪到分发完成的时间)
- 任务执行延迟 (从分发到完成的时间)
- 数据库查询耗时
- 缓存命中率
- 堆内存使用
- GC 频率和停顿时间
